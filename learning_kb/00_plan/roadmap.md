# Roadmap (Godot 4.x)

目标：从零开始，系统学习 Godot 本地 2D 游戏制作能力；Steam 发布不再作为当前主线目标。

## 阶段划分

## Phase 0 - 环境与方法建立（第 1 周）

- 安装 Godot 4.x（稳定版）、Git、基础素材管理工具。
- 熟悉编辑器界面、Scene/Node、Input Map、运行与调试。
- 跑通一个最小可玩原型（移动、跳跃、碰撞、相机、重开）。

## Phase 1 - 核心引擎能力（第 2-5 周）

- 2D 节点体系、信号、状态机、UI、资源管理。
- 存档/设置、项目结构、日志与调试习惯。
- 输出一个可玩 vertical slice（有开始/过程/结束）。

## Phase 2 - 小型完整游戏（第 6-14 周）

- 扩玩法循环（敌人/关卡/经济或分数系统）。
- 完成内容生产管线（场景复用、参数化、数据驱动）。
- 达到 alpha/beta 可测状态。

## Phase 3 - 发布准备（第 15-20 周）

- 性能与稳定性修复、输入适配、分辨率与窗口策略。
- 本地导出与运行包验证（Windows 优先）、崩溃日志与回归清单。
- Steam 商店素材与上线流程降级为远期可选，不纳入当前必修主线。

## 第24课后本地学习路线

- 规划文件：`learning_kb/00_plan/local_godot_learning_curriculum_after_lesson24.md`
- 必修：第25课到第36课，覆盖 Boss 复杂对象、输入、UI、Animation/Tween/Timer、音频、Resource 深化、场景复用、存档设置、调试、性能、本地导出、小项目收口。
- 选修：第37课到第42课，覆盖 Shader、TileMap、手柄输入、简单 AI/寻路、自动化/半自动回归、Godot 插件/Editor 工具脚本。
- 规划原则：按学习能力覆盖组织课程，不按游戏内容堆料。

## 双轨长期规划（学员线 + 导师线）

- 学员线（你亲自学习）
  - 以“课次 runbook”（第1课/第2课/第3课...）为主线，稳定推进可运行项目功能。
  - 每节课固定SOP：回顾 -> 目标 -> runbook -> 验收 -> 修复 -> 收尾。
  - 课次与自然日解耦：一天可完成多课，也可不学习。
  - 核心指标：每周至少 1 个可运行增量。

- 导师线（AI 自学习）
  - 以 docs + 源码证据化学习为主，沉淀模块/卡片/QA/排错模板。
  - 每轮导师学习都必须同步输出“学员线下一节课计划”。
  - 核心指标：知识资产覆盖度 + 导师能力看板百分比。

## 持续性规则

- 每天产出一个“可验证结果”，哪怕很小。
- 每周一次复盘：进度、阻塞、下周计划。
- 每个关键结论尽量附来源（官方 docs / class reference / 源码路径）。
- 若轨道有歧义，默认学员线；切轨必须明确声明。
- 新规则优先写入 `AGENTS.md` + `session_protocol.md` + `00_meta/dual_track_governance.md`，确保下一个 AI 可接管。
