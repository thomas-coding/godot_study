## Key Understanding

- Frame budget is a target envelope; for 60 FPS, one frame has about `16.6ms`.
- A single current `Frame Time` value is insufficient; spikes and curve shape matter.
- Startup sampling can be polluted by loading, `_ready()` work, window initialization, and profiler timing.
- Optimization claims require same route, same sampling window, and before/after evidence.
- A code cleanup can be valuable even when it is not a measurable performance optimization.
