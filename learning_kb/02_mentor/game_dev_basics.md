# Game Dev Basics

## 首作策略（Verified by industry practice, 待补具体来源）

- 优先做小体量 2D，避免首作范围失控。
- 先做 vertical slice（端到端可玩一小段），再扩内容。
- 先打通流程（开始 -> 游玩 -> 结束 -> 重开），再追求表现细节。

## 每日产出定义

- 至少一个可运行变化（功能/修复/流程）。
- 至少一个可解释变化（你能说清楚为什么这样做）。
- 至少一个可复用变化（可沉淀成模板或规则）。

## 技术学习优先级

1. 玩法循环与输入反馈
2. 场景组织与状态管理
3. UI 与数据保存
4. 调试与导出
5. 性能与发布
